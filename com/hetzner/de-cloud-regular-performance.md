### Flexibilität trifft Performance ###

Die Einsatzmöglichkeiten sind extrem vielfältig. Der Tarif eignet sich besonders für Dienste mit überschaubarer Nutzerzahl, wie beispielsweise Familien, Vereine oder kleine Unternehmen. Besonders sinnvoll ist er für Anwendungen mit wechselnder Auslastung, wie beispielsweise Groupware, geteilte Medienspeicher oder private Blogs.

Entwicklerteams können ihn außerdem für Kubernetes-Cluster in Entwicklungs- und Staging-Umgebungen nutzen und neue Features testen, ohne teure Hardware bezahlen zu müssen.

Alles drin, was man braucht.
----------

API

Für die automatische Verwaltung der Cloud-Infrastruktur stehen alle wesentlichen Funktionen auch über eine entwicklerfreundliche REST-API sowie ein CLI-Tool zur Verfügung.

Integrations

Integrationen für diverse Programmiersprachen (z.B. Python und Go) und Management-Frameworks (z.B. Ansible, Terraform oder Kubernetes) erlauben einen hohen Grad der Automatisierung und Integration der Cloud-Infrastruktur beispielsweise in CI/CD-Pipelines.

Networks

Mit Networks können die Server über ein privates Netzwerk kommunizieren. Zudem lassen sich komplexe Netzwerk-Topologien erstellen. Typische Anwendungsfälle sind der Betrieb eines Kubernetes-Clusters oder einer nicht öffentlich erreichbaren Datenbank.

Firewalls

Unsere stateful Firewalls ermöglichen eine einfache Absicherung der Infrastruktur ohne zusätzliche Kosten. Anhand definierter Regeln wird unerwünschter ein- und ausgehender Datenverkehr blockiert. Nach der Konfiguration kannst du sie beliebig vielen Cloud-Servern zuweisen.

Images

Ubuntu, Debian, Fedora und viele weitere Linux-Distributionen stehen als Betriebssystem zur Auswahl. Ob du also eine besonders einsteigerfreundliche Lösung suchst oder langfristig planbare Unternehmensserver betreiben willst - die passende Option ist verfügbar.

Backups sind automatische, tägliche Kopien der Server-Festplatte und ermöglichen die schnelle Wiederherstellung oder das Erstellen eines Duplikats. Häufig gestellte Fragen
----------

* **Wie werden die Cloud Server abgerechnet?**

  Alle Cloud Server haben sowohl eine monatliche Preisobergrenze als auch einen Preis pro Stunde. Die Rechnung deines Cloud Servers wird niemals die monatliche Preisobergrenze überschreiten. Wenn du den Cloud Server vor Ablauf eines Monats löschst, wird dir nur der Stundensatz in Rechnung gestellt. Bis zur Löschung wird dir jeder Cloud Server berechnet, unabhängig davon ob er aktiv genutzt wird.

* **Wie hoch ist die Bandbreite Ihrer Cloud-Server?**

  Die Host-Systeme für unsere Cloud-Instanzen verfügen alle über eine redundante 10-Gbit-Verbindung. Diese Verbindung wird von allen Instanzen auf dem Host gemeinsam genutzt. Wir bieten keine Bandbreitengarantien für unsere Cloud-Server, aber du kannst mit etwa 300 bis 500 Mbit rechnen.

* **Gibt es eine Bibliothek, um von meiner Programmiersprache aus auf Hetzner Cloud zuzugreifen?**

  Wir unterhalten derzeit offizielle Open-Source-Bibliotheken für [Go](https://github.com/hetznercloud/hcloud-go) und [Python](https://github.com/hetznercloud/hcloud-python). Bibliotheken für andere Sprachen, die von der Community erstellt wurden, findest du in unserer [Awesome Hetzner Cloud List](https://github.com/hetznercloud/awesome-hcloud#libraries). Du kannst auch gerne selbst etwas entwickeln! Wirf einen Blick auf unsere API-Dokumentation unter [docs.hetzner.cloud](https://docs.hetzner.cloud/).

* **Wird die Cloud an weiteren Standorten in verschiedenen geografischen Regionen angeboten werden?**

  Wir haben unsere Cloud-Standorte in den vergangenen Jahren kontinuierlich erweitert. Mittlerweile haben wir neben Deutschland und Finnland auch internationale Locations in Ashburn, Virginia; Hillsboro, Oregon (USA) sowie Singapur. Wir prüfen weiterhin mögliche Optionen. Wenn Sie die Eröffnung eines neuen Standorts nicht verpassen wollen, abonnieren Sie unseren [Newsletter](https://www.hetzner.com/de/newsroom/newsletter/) oder folgen Sie uns auf unseren Social-Media-Kanälen.

* **Wann wird Feature X kommen?**

  Wir wollen uns Produkte immer weiter verbessern und durch neue Funktionen und Dienste ergänzen. Dazu sind wir auf dein Feedback angewiesen. Es gibt mehrere Möglichkeiten, wie du uns Produkt-Feedback oder Funktionswünsche mitteilen kannst:

  * Über unser Forum: Bitte poste Feedback im [Hetzner Cloud Board](https://forum.hetzner.com/index.php?board/117-hetzner-cloud/)
  * Per E-Mail: Sende deine Ideen an [product-feedback@hetzner.com](mailto:product-feedback@hetzner.com)

  Leider können wir dir keine genaue Roadmap der Funktionen und Dienste, an denen wir arbeiten, zur Verfügung stellen. Du kannst uns aber auf unseren Social-Media-Kanälen folgen und so auf dem Laufenden bleiben.

* **Kann ich Windows installieren?**

  Derzeit konzentrieren wir uns auf die Unterstützung von Linux als Betriebssystem. Wenn du möchtest, kannst du Windows selbst installieren. Wir haben bereits erfolgreiche Installationen gesehen. Allerdings bieten wir keinen Support für Windows an.

* **Welche Linux-Distributionen werden unterstützt?**

  Hetzner unterstützt eine Reihe gängiger Linux-Distributionen: Ubuntu, Fedora, Debian, CentOS, Rocky Linux und AlmaLinux. Du kannst diese bei der Erstellung von Cloud-Servern auswählen oder auf dedizierten Servern vorinstallieren lassen. Außerdem kann jedes andere Betriebssystem manuell über KVM/ISO installiert werden.
